# SkyBrain 解决方案 V2 — 脑洞探索：MCP 对话式无摩擦知识录入

> 状态：讨论完成
> 阶段：V2 解决方案与产品雏形
> 核心问题：传统 IDE 插件右键录入太重，开发者不愿配合。如何在写代码的自然流中，零额外动作完成知识沉淀？

---

## 一、 问题缘起：谁会有耐心专门去"录知识"？

前端议题确立了 SkyBrain 需要源源不断的知识录入来喂养大脑。但现实很残酷：

> 一线开发者最讨厌的就是"额外操作"。让他们写完代码后，专门打开某个界面、切换窗口、填表单、手工整理"我学到了什么"——这在工程实践中基本无法推行。

传统的知识管理工具（Wiki、Notion、Confluence）之所以在研发团队中变成"文档坟场"，根本原因就是**录入的摩擦力太大**。SkyBrain 如果不能从根上解决这个问题，就会重蹈覆辙。

**核心矛盾**：知识沉淀的"整理成本"与开发者"有限的心智带宽"之间的天然冲突。

---

## 二、 核心设计理念：把"录入"变成"顺嘴说一句"

经过探讨，我们果断放弃了传统的"IDE 右键菜单上传"路线，核心理由：

- 主流 AI IDE 繁多（Cursor、Trae、Copilot、Cline 等），插件适配成本极高
- 就算做了插件，开发者还需要"选中 → 右键 → 填写 → 保存"，仍然是额外操作
- 插件生态变化快，维护成本会被无限放大

取而代之的核心理念：

> **让开发者完全不需要离开写代码的上下文。** 在 AI 对话中，像跟同事讨论问题一样自然表达，SkyBrain 自动识别并完成知识沉淀。

---

## 三、 最终方案：MCP 对话式反向写入

交互形态极其简单——开发者在 AI 对话框中说一句人话：

> "把这段重试逻辑作为开发规范，保存到 SkyBrain"

底层发生了什么：

1. **MCP 暴露写入型工具**：SkyBrain Local Daemon（作为 MCP Server）向 IDE 暴露 `save_to_skybrain(content, tags)` 这样的"写入型"工具
2. **大模型理解意图**：大模型从开发者的自然语言中识别出"这是一个保存知识"的意图
3. **大模型调用工具**：大模型自动调用 `save_to_skybrain`，将梳理好的规则内容（格式化后的代码 + 标签 + 说明）反向推送给本地客户端
4. **本地沙盒存储**：Local Daemon 接收后存入开发者的个人沙盒，不直接污染中央大脑

**最关键的创新在于方向的反转**：传统 MCP 是"AI 从外部拉数据"，SkyBrain 的创新是"AI 往外部写数据"。MCP 协议本身没有限制方向，只是大家习惯性把它当"只读工具"，这层窗户纸被戳破了。

---

## 四、 方案面临的核心挑战与应对策略

### 挑战 1：大模型"理解偏了"

**问题**：开发者说"保存一下"，大模型可能错误解读了范围——保存了整个对话而不是特定的那段代码；或者标签打错了（把"数据库规范"标成了"缓存策略"）。

**应对**：
- `save_to_skybrain` 工具设计了必填的结构化参数：`content`（内容）、`tags`（标签）、`scope`（项目/模块范围），大模型必须填对了才能调用成功
- AI 写入后在 MCP 对话中给一个简短的确认摘要："已将《UserService 重试逻辑》保存至你的本地沙盒，已标注标签：Java、Retry、Resilience。需要时可以说'把这提报给团队'来提交审核。"

### 挑战 2：开发者"忘了说"——最有价值的知识恰恰没被录

**问题**：最宝贵的知识往往是开发者默默解决的坑——他花了 2 小时才搞明白某个诡异行为，修完后身心俱疲直接下班，完全忘了"保存到 SkyBrain"。

**应对**：
- 这是 MCP 反向写入的固有盲区——它是"被动触发"的，依赖开发者主动开口
- 需要通过**自动化扫描（议题二的 Channel B）**作为补充——系统自动分析代码库提炼知识，不依赖开发者开口
- 也可以在 Local Daemon 中做"战后回顾"——当一个会话结束后检测到大量手动修改，提示"这次你似乎花了很多时间解决这个问题，要保存为知识吗？"

### 挑战 3：自然语言的歧义性

**问题**：开发者可能用语模糊——"把这个也存一下"、"跟上次一样处理"——大模型缺乏上下文来理解"这个"和"上次"。

**应对**：
- MCP 工具参数设计包含上下文锚点，如 `current_file`、`selected_code`、`session_topic`，由 Local Daemon 自动填充，避免大模型猜错
- 遇到无法确定的指令，AI 主动追问而非瞎猜

## 六、 结论与后续行动

本次脑洞探索明确了一个关键创新：

1. **方向反转**：MCP 不只是"只读查询通道"，SkyBrain 将它反向用于"写入"，实现了完全在心流中的知识沉淀
2. **零摩擦力**：开发者不需要打开任何新窗口、不需要点击任何按钮、不需要切换任何工具——顺嘴说一句就搞定
3. **与三级生命周期联动**：MCP 写入的目标是"个人沙盒"而非"中央大脑"，既保证了录入的极简体验，又不污染全局知识库

这个脑洞是整个 SkyBrain 知识录入体系的**交互基础**，后续的三级生命周期、知识 GC 等机制都建立在这个"无摩擦入口"之上。
